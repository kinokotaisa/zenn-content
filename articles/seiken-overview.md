スマホの画面に並んだボタンを、ひとつタップする。

すると数メートル離れた机の上で、人形が「シュッ」と正拳突きを繰り出した。画面のカウンターが `+1` される。

見た目はポップなおもちゃ。でも中身は、自作基板・ファームウェア・Webアプリ・ネットワークが一本につながった、れっきとしたシステムだ。この記事では、その「かわいい見た目」と「まじめな中身」のギャップを、まるごと紹介していく。

## なぜ作っているのか

ネット通販でポチった商品が翌日には届く物流、駅の改札、話しかけると答えるスマートスピーカー——普段あたりまえに使っているものほど、「これ、中ではどう動いているんだろう」と気になることはないでしょうか。私はよく思います。

とはいえ、いきなり改札は作れません。でも、何かを「スマホから動かしたい」「動いた回数を記録・解析してみたい」——そんな思いが、このプロジェクトの出発点です。

ハードウェアを組み、ファームウェアで動かし、ネットワークでソフトウェアとつないで、もっと便利にしていく。**HW・FW・SW・NW を統合して、ひとつのシステムとして動くもの**を、作りながら一緒に学んでいく——それがこの連載です。

> 💡 **こんな人と一緒に辿れたら**
> 自分の専門の外に、一歩踏み出してみたい人。「Webはやるけど電子工作は未知」「組み込みはやるけどサーバー側は触ったことがない」——その境界を、実際に動くデバイス1台で一望してみます。

その最初の一台が、冒頭の正拳突き人形です。スマホのタップが命令となって人形に届き、繰り出された**正拳**をラズパイのカメラが画像認識で数える——指先のひと押しが画面のカウントに変わるまで、信号が層から層へどう渡っていくのかを、このあと実物で追いかけます。

## このプロジェクトの地図

### まず、身近なシステムの「形」

さきほど挙げたスマートスピーカーを思い出してください。「明日の天気は？」と聞くと答えが返ってくる。でもあの返事は、手のひらのスピーカーの中だけで作られているわけではありません。あなたの声はネット越しに**遠くのサーバー**へ送られ、そこで処理されて、また戻ってきます。

つまりこのシステムは、**手元のモノ ── ネットワーク ── 遠くのサーバー** の3点でできている。現場で入力・検知し、サーバーが処理・記録し、また現場へ返す。私たちが普段「便利」と感じているものの多くは、だいたいこの形をしています。

### その「形」を、自分でつくる

だったら、同じ形を自分でも作れるはずです。

自分で組んだデバイスをネット越しにサーバーへつなぎ、ブラウザから動かしたり、動いた回数を記録してグラフにしたり。そして、デバイスが増えてきたら、同じ土台にどんどん挿していける——そんな「**自分専用の土台**」をつくる。これがこのプロジェクトの中心にあります。

その名も「**seikenシステム**」。さっきのスマートスピーカーと、形はまったく同じです。違うのは、つながっているのが市販品ではなく、自分で作ったデバイスだということ。

| さっきの3点 | seikenシステムでは |
| :--- | :--- |
| 手元のモノ | **デバイス**（正拳突き人形、水やりぞうさん…） |
| ネットワーク | インターネット越しの通信 |
| 遠くのサーバー | **seikenサーバー**（ブラウザからの遠隔操作＋データの記録・可視化） |

```
        スマホ / PC ブラウザ
               │
     seiken サーバー（遠隔操作・データ記録）
               │   ← ネットワーク
   ┌───────────┼───────────┐
正拳突き人形   水やりぞうさん   …これから増える
```

ブラウザのダッシュボードには登録した機器がカードで並び、選ぶと遠隔操作・カメラ映像・グラフが見られます。

いまはまだ、すべてがラズパイ1台の上で動く「ローカル完結型」。土台に乗っているのは、第一号の正拳突き人形だけです。次の章では、スマートスピーカーと同じこの「形」が、正拳突き人形では具体的にどう実現されているのか——HWからNWまで縦に辿ります。

## 【実例】正拳突き人形が動くまでを、縦に辿る

ここからが本題です。冒頭の「スマホをタップ → 人形が正拳 → カウント」を、HW・FW・SW・NW の4層に分解して、下から上へ辿ってみます。

### HW（ハードウェア）— 動く体をつくる

人形の体は、3Dプリントしたフレームに、2個のサーボモーター（SG90）と「スライダクランク機構」を仕込んだもの。サーボの回転が腕の直線運動に変わり、正拳突きになります。頭脳は手のひらサイズのマイコン **ATtiny402**（1個80円）。ここに圧力センサ・手押しボタン・音声モジュール（DFPlayer Mini＋スピーカー）がつながり、単3電池3本（4.5V）だけで動く。電源を入れれば単体で完結する「スタンドアロンのおもちゃ」が、ここで一旦完成します。

### FW（ファームウェア）— 体に動き方を教える

ATtinyに書き込むプログラムがファームウェアです。やっていることはシンプルで、「**トリガーが来たら、音を鳴らして、ワンツーパンチを3回**」。圧力センサが押される（ボールが当たる）かボタンが押されると、サーボを引き位置(150°)→突き位置(30°)と振り、1セット約2.4秒のパンチを繰り出します。ここまでは、まだサーバーもネットワークもいらない世界です。

### SW（ソフトウェア）— スマホから動かし、記録する

ここで seikenシステムが登場します。ブラウザ側（Vue.js）には遠隔操作ボタン、正拳回数モニター（今日・累計・グラフ）、カメラ映像が並ぶ。サーバー側（Flask）が指示を受けて結果をデータベースに記録し、ラズパイ上のプログラムがカメラ映像を **OpenCVで画像認識**して、人形が実際にパンチした回数を数えます。**遠隔モードで回数を数えるのは、人形のセンサーではなく、見ているカメラ**です。

### NW（ネットワーク）— 4層をひとつにつなぐ

最後に、これらを1本の線でつなぐのがネットワークです。スマホのタップは **Socket.IO** という仕組みで、サーバーへ、そしてラズパイへとリアルタイムに伝わります。

そして、いちばん面白い"つなぎ目"がここにあります。ラズパイは、人形の **圧力センサと同じ端子** に「押された」のと同じ電気信号（LOWパルス）を送り込むだけ。ATtinyから見れば、ボールが当たったのと区別がつきません。**だから遠隔操作のために、ファームウェアは1行も書き換えていない**。「ボール検知で起動」も「Webから起動」も、人形にとっては同じこと。スタンドアロンのおもちゃが、配線一本で**ネットワーク機器に大変身！！**

```
[スマホ] ──タップ──► [サーバー(Flask)] ──Socket.IO──► [ラズパイ]
                                                          │ GPIO LOWパルス
                                                          ▼
                                                 [ATtiny402] → 正拳突き
                                                          │
[ブラウザに反映] ◄─ [サーバーで記録(DB)] ◄─ [カメラ＋OpenCVが回数を数える]
```

タップひとつが、サーバーを通り、電気信号になって人形を動かし、その動きをカメラが見て数え、また画面に戻ってくる。HW・FW・SW・NW が、こうしてひとつのループになります。次の章からは、この一周を支える技術を、連載で順番に掘り下げていきます。

## これからの道のり

正拳突き人形は、この長い旅の「最初の一台」です。ここから先、このプロジェクトは2つの方向に広がっていきます。さきほどの地図でいう、**土台（seiken）を育てる軸**と、**デバイスを増やす軸**です。

### 🌐 土台を育てる：seikenシステムの道のり

いまの seiken は、すべてが1台のラズパイに同居した「ローカル完結型」。ここから、本物のシステムらしく一歩ずつ育てていきます。

| 道のり | やること |
| :-- | :-- |
| **① ローカルで完成させる**（←いまここ） | まず手元で、1台に同居した形をしっかり動かしきる |
| **② クラウドに載せる** | インターネット越しに、どこからでも使えるようにする |
| **③ いろんなデバイスを載せられる土台へ** | 特定のおもちゃ専用ではなく、何でも挿せる「プラットフォーム」にする |

ステップが進むたびに、新しい技術の"つなぎ目"が出てきます。その都度この連載で、つまずいた所も含めて辿っていきます。

### 🤖 土台に乗せるデバイスたち

土台を育てるのと並行して、そこに乗せるデバイスも作っていきます。

| デバイス | いま |
| :-- | :-- |
| 正拳突き人形 | 🟢 seiken統合済み・運用中 |
| 水やりぞうさん | 🔬 試作中（ESP32）。次のseiken統合候補 |
| ライントレーサー | 💡 構想中 |
| セグウェイ風の乗り物 | 💡 構想中 |

おもしろいのは、デバイスごとに"つなぎ方"が変わること。たとえば水やりぞうさんは Wi-Fi 内蔵の ESP32 なので、正拳突き人形のようにラズパイを挟まず、サーバーへ直接つなげます。同じ土台でも、デバイスが違えば道のりも変わる——そのバリエーションも、これから紹介していきます。

## どう読んでほしいか

最後に、この連載の歩き方を少しだけ。

この記事は、この連載全体の入口です。これから Step やデバイスごとに記事が増えていきますが、どれも今日見た「タップ → 動く → 数える」ループのどこかを掘り下げたもの。だから、頭から順に読む必要はありません。気になった層・気になったデバイスから、つまみ食いしてください。

冒頭でも触れたとおり、これは「教える」連載ではなく、つまずきも含めて一緒に辿っていく記録です。気軽に隣で覗いているくらいの距離感で読んでもらえたら嬉しいです。

発信は、2つの形でやっていきます。

- 🎬 **ショート動画** … 完成したデバイスが動く様子を、サクッと楽しく
- 📝 **この連載（Zenn）** … その裏側、HW〜NWの"つなぎ目"をじっくり

「動いてる！」を入口に、「どうやって動くの？」まで付き合ってもらえたら最高です。次の記事からは、この地図を一マスずつ歩いていきます。
